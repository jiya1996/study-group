# 协作规范

## 分支策略

- `main` 分支保持稳定
- 新内容用 `feature/xxx` 分支，完成后 PR 合并
- daily-digest 由自动化任务直接提交到 main

## 提交规范

```
类型: 简短描述

类型包括：
- 新增：新内容/新文件
- 更新：修改已有内容
- 整理：目录结构调整
- 修复：错误修正
```

## 内容组织

- 产品相关文档放 `docs/` 对应子目录
- 学习笔记放对应主题目录
- 每日资讯由自动化生成到 `daily-digest/`

## 隐私注意

- 不要提交个人信息（API Key、服务器地址、个人联系方式等）
- 提交前检查 `.gitignore` 确保敏感文件被排除
